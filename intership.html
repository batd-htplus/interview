4. Coding

Estimation (Dự tính)

Không nhận task xong làm ngay. Phải chia nhỏ task, dự tính thời gian (worst case) và commit deadline với Mentor/PM.

Log Bug & Báo cáo

Ghi rõ: Steps to reproduce, Expected vs Actual result. Báo cáo tiến độ hàng ngày qua Standup (Yesterday - Today - Blockers).

Viết QAs

Không tự đoán requirement! Nếu ambiguous, phải viết câu hỏi rõ ràng (QAs) gửi PM/Team trước khi code để tránh làm lại.

Văn hóa Code Review

Không coi Code Review là bị soi. Đó là quá trình học hỏi từ Senior. Luôn giải thích rõ ràng logic trong Merge Request.

ISMS & Nội quy Bảo mật Công ty

Sử dụng Phần mềm & Thiết bị

Chỉ dùng phần mềm có phép. Không cài crack. Không truy cập trang độc hại.

Xử lý Virus/Malware

Ngắt mạng ngay. Báo qua Rocket.Chat (IT Support). Không tự xóa file.

Bảo mật Source Code

Không push public repo. Không lộ token/API key ra ngoài.

Bảo mật Tài liệu

Không gửi tài liệu nội bộ (design, schema) ra email cá nhân. Tuân thủ NDA.

Kết quả Cuối cùng (Sau 12 Tuần)

Kỹ Năng Kỹ Thuật

  • Build REST API production-ready
  • Design DB index đúng, tránh N+1
  • Viết test ≥80% coverage
  • Build FE kết nối BE thật
  • Containerize bằng Docker
  • Deploy Staging & xử lý incident

Về Tư Duy

  • Xóa mindset "chạy là được"
  • Biết tự debug thay vì hỏi ngay
  • Hiểu tại sao code phải maintainable
  • Có security awareness cơ bản
  • Biết đọc & review code người khác

Thực Tế Làm Việc

  • Nhận task thật trong team
  • Tự implement feature end-to-end
  • Tự tạo PR, self-review, fix feedback
  • Không cần mentor cầm tay từng bước
